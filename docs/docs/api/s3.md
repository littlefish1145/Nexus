# S3-Compatible API Reference

Nexus provides an S3-compatible API that supports the most commonly used
AWS S3 operations.

## Base URL

```
http://<nexus-host>:9000
```

## Authentication

All requests must be signed using AWS Signature Version 4 (SigV4).

### Headers

| Header | Description |
|--------|-------------|
| `Authorization` | AWS4-HMAC-SHA256 credential signature |
| `X-Amz-Date` | Request timestamp in ISO 8601 format |
| `X-Amz-Content-SHA256` | SHA256 hash of the request payload |

## Bucket Operations

### CreateBucket

```
PUT /{bucket}
```

Creates a new bucket.

**Response**: `200 OK`

### DeleteBucket

```
DELETE /{bucket}
```

Deletes an empty bucket.

**Response**: `204 No Content`

### ListBuckets

```
GET /
```

Lists all buckets owned by the authenticated user.

**Response**:

```xml
<ListAllMyBucketsResult>
  <Owner>
    <ID>owner-id</ID>
    <DisplayName>owner-name</DisplayName>
  </Owner>
  <Buckets>
    <Bucket>
      <Name>my-bucket</Name>
      <CreationDate>2024-01-01T00:00:00.000Z</CreationDate>
    </Bucket>
  </Buckets>
</ListAllMyBucketsResult>
```

### HeadBucket

```
HEAD /{bucket}
```

Checks if a bucket exists.

**Response**: `200 OK` or `404 Not Found`

## Object Operations

### PutObject

```
PUT /{bucket}/{key+}
```

Uploads an object.

**Headers**:

| Header | Description |
|--------|-------------|
| `Content-Type` | MIME type of the object |
| `x-amz-meta-*` | Custom metadata |
| `x-amz-server-side-encryption` | Encryption method (`AES256` or `aws:kms`) |
| `x-amz-server-side-encryption-customer-algorithm` | SSE-C algorithm (`AES256`) |
| `x-amz-server-side-encryption-customer-key` | SSE-C encryption key (Base64) |
| `x-amz-server-side-encryption-customer-key-MD5` | SSE-C key MD5 (Base64) |

**Response**: `200 OK`

### GetObject

```
GET /{bucket}/{key+}
```

Downloads an object.

**Headers** (for SSE-C):

| Header | Description |
|--------|-------------|
| `x-amz-server-side-encryption-customer-algorithm` | `AES256` |
| `x-amz-server-side-encryption-customer-key` | Customer key (Base64) |
| `x-amz-server-side-encryption-customer-key-MD5` | Key MD5 (Base64) |

**Response**: `200 OK` with object body

### HeadObject

```
HEAD /{bucket}/{key+}
```

Retrieves object metadata without the body.

**Response**: `200 OK` with headers only

### DeleteObject

```
DELETE /{bucket}/{key+}
```

Deletes an object.

**Response**: `204 No Content`

### ListObjectsV2

```
GET /{bucket}?list-type=2
```

Lists objects in a bucket.

**Query Parameters**:

| Parameter | Description |
|-----------|-------------|
| `prefix` | Filter by key prefix |
| `delimiter` | Group keys by delimiter |
| `max-keys` | Maximum number of keys (default 1000) |
| `continuation-token` | Token for pagination |
| `start-after` | Start listing after this key |

### CopyObject

```
PUT /{bucket}/{key+}
x-amz-copy-source: /source-bucket/source-key
```

Copies an object.

## Multipart Upload Operations

### CreateMultipartUpload

```
POST /{bucket}/{key+}?uploads
```

Initiates a multipart upload.

**Response**:

```xml
<InitiateMultipartUploadResult>
  <Bucket>my-bucket</Bucket>
  <Key>large-file.bin</Key>
  <UploadId>upload-id</UploadId>
</InitiateMultipartUploadResult>
```

### UploadPart

```
PUT /{bucket}/{key+}?partNumber={n}&uploadId={id}
```

Uploads a part of a multipart upload.

**Response**: `200 OK` with `ETag` header

### CompleteMultipartUpload

```
POST /{bucket}/{key+}?uploadId={id}
```

Completes a multipart upload.

**Body**:

```xml
<CompleteMultipartUpload>
  <Part>
    <PartNumber>1</PartNumber>
    <ETag>"etag-1"</ETag>
  </Part>
  <Part>
    <PartNumber>2</PartNumber>
    <ETag>"etag-2"</ETag>
  </Part>
</CompleteMultipartUpload>
```

### AbortMultipartUpload

```
DELETE /{bucket}/{key+}?uploadId={id}
```

Aborts a multipart upload.

## Error Responses

All errors follow the S3 error format:

```xml
<Error>
  <Code>NoSuchBucket</Code>
  <Message>The specified bucket does not exist.</Message>
  <Resource>/my-bucket</Resource>
  <RequestId>req-id</RequestId>
</Error>
```

### Common Error Codes

| Code | HTTP Status | Description |
|------|-------------|-------------|
| `AccessDenied` | 403 | Insufficient permissions |
| `BucketAlreadyExists` | 409 | Bucket name already taken |
| `BucketNotEmpty` | 409 | Cannot delete non-empty bucket |
| `InvalidSSECustomerKey` | 400 | Invalid SSE-C key |
| `NoSuchBucket` | 404 | Bucket does not exist |
| `NoSuchKey` | 404 | Object does not exist |
| `SignatureDoesNotMatch` | 403 | Invalid request signature |
| `SlowDown` | 503 | Rate limit exceeded |

## Nexus Extensions

### Vector Search

```
POST /_vector_search
Content-Type: application/json

{
  "query": "search query text",
  "top_k": 20
}
```

### Full-Text Search

```
POST /{bucket}/_fts
Content-Type: application/json

{
  "query": "search terms",
  "top_k": 10
}
```

### Resumable Upload

```
POST /{bucket}/_resumable
Content-Type: application/json

{
  "key": "large-file.bin"
}
```
